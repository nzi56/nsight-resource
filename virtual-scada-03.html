SCADA SIMULATOR MODE

Outage Simulation & Restoration Assistant

Below is a comprehensive expansion on the Outage Simulation & Restoration Assistant module—designed to turn our WebGIS tracing engine into a pre-SCADA outage command center. Think of it as a "SCADA Simulator Mode" that utilities in Bangladesh (or any emerging market) can deploy today using only GIS + basic customer data.

Module Goal

Allow control room staff, planners, and field supervisors to simulate any outage scenario, instantly see impacts, and receive AI-guided restoration steps—all inside the WebGIS platform.

1. Interactive Outage Creation (Drag-Drop or Click)

Create Any Fault or Planned Outage Scenario

Operators can inject faults, open switches, model storm damage, or schedule maintenance directly on the map, without affecting the live GIS data.

Action How It Works
Fault a Device Right-click any line, transformer, or switch → "Simulate Fault Here" → segment turns red and is isolated.
Open a Switch Click switch symbol → toggle to OPEN (visual: red X). System auto-recomputes connectivity.
Bulk Storm Mode Draw polygon → "Apply Storm Damage" → randomly opens 10–30% of devices inside (configurable).
Scheduled Maintenance Select devices → "Plan Outage: 2 Nov, 08:00–12:00" → system locks state for that window.

All changes are non-destructive—saved as "Scenarios" (similar to Git branches) that can be cloned, compared, or archived.

2. Instant Downstream Impact Analysis (Auto-Trace)

Impact in Seconds

As soon as a fault is simulated or a switch is opened, the system auto-traces downstream and calculates electrical, customer, and critical load impact.

Output Visual Data
De-energized Network Blinking red trace Full electrical path downstream
Affected Customers Heatmap + count badge Linked to CIS / billing DB (by transformer / meter)
Critical Loads ⭐ Hospital / Factory / BTS icons blink Pre-tagged in GIS
Estimated Load Off Σ 1.4 MW From transformer nameplate or AMI average
Outage Duration Prediction “Est. 2.5 hrs” Based on crew proximity + historical fix time

Example: Fault on 11kV Feeder F6 → 1,240 customers, 3 hospitals, 1.1 MW off → all impacted objects blink on the map in < 2 seconds.

3. AI-Powered Restoration Wizard (Step-by-Step)

From Fault to Optimal Reconfiguration

Click "Restore" and the system runs a network reconfiguration algorithm to propose the most efficient restoration path.

RESTORATION PLAN (3 steps, 96% restoration)

Step 1: Close Tie-Switch TS-12 (Feeder F5 → F6)
    → Restores 920 customers (74%)
    → Load on F5: 68% → SAFE

Step 2: Close Backup Switch BS-03 (Substation B)
    → Restores 300 more (total 98%)
    → Creates loop → Auto-opens NOP-7 to isolate

Step 3: Isolate faulted section (open SW-44 & SW-45)
    → Faulted line: L6-SEG3 (200m)

Estimated Crew Time: 2.1 hours
  • Each step is click-to-execute in simulation (or manually applied in the field).
  • Overloaded feeders (> 90% loading) are highlighted in orange.
  • Flags protection miscoordination (e.g., "Fuse F12 may save breaker CB-6").

5. Integration with Field Crews (Mobile Sync)

Closing the Loop Between Control Room & Field

Scenarios are pushed to field tablets via the WebGIS mobile app, ensuring crews and control room work from the same playbook.

  • Push scenario maps and step-by-step instructions to field crews.
  • Crew taps: "Switch TS-12 closed at 14:32" → system updates live simulation.
  • Auto-retrace confirms 920 customers restored.
  • Crew sees the next step highlighted on the map in real time.
