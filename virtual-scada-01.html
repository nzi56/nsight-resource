VIRTUAL / LOGICAL SCADA

Mimicking Virtual / Logical SCADA Functionality

Our WebGIS-based network tracing application effectively serves as a virtual or logical SCADA system by leveraging the geospatial topology of the electrical network to simulate real-time operational awareness— without requiring physical sensors, RTUs, or telemetry infrastructure.

"Virtual SCADA through GIS Topology Intelligence"

Positioning Statement
Narrative for Presentations / Proposals / Documentation

In the absence of a physical SCADA system, this WebGIS application transforms static network data into a dynamic, logical SCADA-like control room directly within the browser. By initiating an upstream or downstream trace from any selected connection point—be it a feeder, transformer, switch, or customer meter—the system instantly computes and visualizes the energized electrical path in real time, mimicking SCADA’s core supervisory functions:

Downstream Network

  • Real-time network state inference: Traces respect open/closed states of switches and protection devices (if modeled), simulating isolation zones, fault sections, and supply sources—just like SCADA alarm processing.
  • Dynamic visual alarming via blinking layers: Critical traced elements (e.g., overloaded segments, fault-isolated zones, or de-energized customers) can trigger blinking animations, replicating SCADA’s visual alert system for immediate operator attention.
  • Logical telemetry simulation: Instead of polling field devices, the system uses GIS connectivity rules to logically derive power flow direction, affected downstream load, and restoration paths—providing SCADA-equivalent situational awareness using only digital twin data.

This creates a "SCADA before SCADA" environment—delivering 80% of operational visibility at 20% of the cost and complexity, using existing GIS assets.

Additional High-Value Functionalities to Enhance "Virtual SCADA"

Roadmap to a Pre-SCADA Operations Platform

To evolve this from a tracing tool into a full pre-SCADA operations platform, consider adding:

1. Outage Simulation & Restoration Assistant

  • Let users "open" a switch or "fault" a line → auto-trace the de-energized zone.
  • Suggest optimal switching sequence to restore non-faulted sections.
  • Estimate customers affected (linked to billing database).

2. Load Profile Overlay on Trace Results

  • Overlay estimated or historical load (kW) per transformer / feeder.
  • Highlight segments exceeding 80% loading during trace.

3. Mobile Field Crew Integration

  • Push trace results to field tablets.
  • Crew members tap "I opened this switch" → system updates logical state → re-traces automatically.

4. Automated Daily / Storm Reports

  • Scheduled traces from all HV feeders → auto-generate PDF or email reports such as:
  • "Feeder X: 1,240 customers, 2 open ties, 1 normally open point"

5. Integration with Complaint Management

  • Customer calls "no power" → geocode the address → auto-trace upstream.
  • Show probable cause (e.g., "Open switch at Pole 45").

6. Protection Coordination Viewer

  • During trace, highlight relay / breaker pairs.
  • Flag miscoordination risks (e.g., downstream fuse clearing before upstream breaker).

7. Historical Trace Replay

  • Store switch operations with timestamps.
  • Replay network state evolution over the past 24 hours (similar to a SCADA event log).

8. Integration with AMI Last-Gasp Events

  • When AMI meters send "last gasp" (power off), trigger auto-trace from those endpoints.
  • Instantly map outage boundary based on meter events.
